# File Browser Plugin Contract

Ember exposes the file browser through manifest contributions and EventBus hooks.

## Contributions

```json
{
  "permissions": ["file:read", "file:write", "file:delete", "file:actions"],
  "contributes": {
    "fileActions": [
      {
        "id": "myPlugin.copyImportPath",
        "title": "Copy Import Path",
        "command": "myPlugin.copyImportPath",
        "icon": "copy",
        "group": "200_language",
        "when": "isFile and ext == '.py'",
        "appliesTo": ["file"],
        "permissions": ["file:read"]
      }
    ],
    "fileDecorations": [
      {
        "id": "myPlugin.generated",
        "title": "Generated file",
        "badge": "GEN",
        "color": "#A78BFA",
        "tooltip": "Generated by a tool",
        "when": "isFile and matches('.generated.py')",
        "appliesTo": ["file"]
      }
    ],
    "fileIconOverrides": [
      {
        "id": "myPlugin.protoIcon",
        "pattern": ".proto",
        "icon": "protobuf",
        "appliesTo": ["file"],
        "priority": 700,
        "when": "isFile"
      }
    ]
  }
}
```

## Context Payload

File browser actions, decorations and icon overrides receive:

```json
{
  "path": "/workspace/app/main.py",
  "name": "main.py",
  "isDir": false,
  "type": "file",
  "extension": ".py",
  "workspace": "/workspace",
  "parent": "/workspace/app"
}
```

## Supported `when` Clauses

`when` is intentionally declarative and safe. Supported clauses:

- `isFile`
- `isFolder`
- `ext == '.py'`
- `extension == '.json'`
- `name == 'package.json'`
- `basename == '.env'`
- `matches('.generated.py')`
- Combine clauses with `and` or `&&`.

## File Operation Hooks

Plugins can listen to operation hooks through `api.events.on(...)`.

Blocking hooks are synchronous and may return `{"ok": false, "message": "Reason"}`:

- `file:willCreate`
- `folder:willCreate`
- `file:willRename`
- `folder:willRename`
- `file:willDelete`
- `folder:willDelete`
- `file:willMove`
- `folder:willMove`
- `file:willPaste`
- `folder:willPaste`
- `file:willFormat`

Post-operation events are emitted after success/failure:

- `file:created`, `folder:created`
- `file:renamed`, `folder:renamed`
- `file:deleted`, `folder:deleted`
- `file:moved`, `folder:moved`
- `file:pasted`, `folder:pasted`
- `file:formatted`
- `file:*Failed`, `folder:*Failed`

## Backend Example

```python
def activate(api):
    def block_secret_files(path: str, name: str = "", **_):
        if name.endswith(".secret"):
            return {"ok": False, "message": "Secret files are protected"}

    def copy_import_path(context: dict, action: dict | None = None):
        path = context["path"]
        # Return {"ok": true/false, "message": "..."} for UI notifications.
        return {"ok": True, "message": f"Handled {path}"}

    api.events.on("file:willDelete", block_secret_files)
    api.commands.register("myPlugin.copyImportPath", copy_import_path)
```
